# Shared dependencies for both PCs.
# The operator PC needs all of them: pip install -r requirements.txt
#
# The ROBOT PC only strictly needs eclipse-zenoh (+ opencv-python if it has
# no working system OpenCV already). DO NOT blindly `pip install -r
# requirements.txt` for a venv on the robot: the generic opencv-python PyPI
# wheel is built WITHOUT GStreamer and its V4L2 backend has failed outright
# ("can't open camera by index") on at least one UVC camera we use, while
# the apt-installed system OpenCV (GStreamer-enabled) opens the same camera
# fine. On the robot, create the venv with --system-site-packages and only
# `pip install eclipse-zenoh` so it inherits the working system cv2 instead
# of shadowing it with a broken one:
#     python3 -m venv --system-site-packages .venv
#     .venv/bin/pip install eclipse-zenoh
